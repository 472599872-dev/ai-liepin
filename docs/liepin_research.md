# 猎聘网站研究结论

## 登录与账号

- 猎聘招聘端入口：`https://h.liepin.com/search/getConditionItem`。
- 未登录会跳转到：`/account/login?backurl=...`。
- 登录页默认是短信验证码登录，也有 `密码登录` 切换。
- 密码登录表单：
  - 手机/邮箱输入框：`input[placeholder='请输入认证的手机号/邮箱']`
  - 密码输入框：`input[placeholder='请输入登录密码']`
  - 登录按钮：`登 录`
- 实测密码登录会触发滑块拼图验证。
- 系统不应自动绕过滑块/验证码，应将任务暂停为 `等待人工验证`。
- 账号设计应使用独立 Profile：
  - 每个账号手动验证一次。
  - 登录态保存在本地 Profile。
  - 后续任务复用 Profile，不频繁输入密码。

## 找人页

- 找人页 URL：`https://h.liepin.com/search/getConditionItem`。
- 主要筛选项包括：
  - 包含全部关键词
  - 职位名称
  - 公司名称
  - 目前城市
  - 期望城市
  - 工作年限
  - 教育经历
  - 统招要求
  - 院校要求
  - 当前行业
  - 当前职位
  - 年龄
  - 活跃度
  - 性别
  - 跳槽频率
  - 期望年薪
  - 目前年薪
  - 期望行业
  - 期望职位
  - 毕业院校
  - 专业名称
  - 海外工作经验
  - 管理经验
- 列表页有候选人摘要卡片，但不是完整简历。
- 候选人卡片常见容器：
  - `.detail-resume-card-wrap`
  - `.tlog-common-resume-card`
- 候选人卡片中的 `立即沟通` 是沟通动作，不应作为简历入口点击。

## 简历详情页

- 点击候选人卡片会新开详情页，不是在当前列表页原地跳转。
- 详情页 URL 形态：`https://h.liepin.com/resume/showresumedetail/?res_id_encode=...`
- 应用必须接管新窗口/新标签请求，否则用户会觉得“卡片打不开”。
- 详情页正文可通过 DOM 读取：`document.body.innerText`。
- 详情页可命中的主要区块：
  - 简历信息
  - 求职意向
  - 工作经历
  - 项目经历
  - 教育经历
  - 资格证书
  - 语言能力
  - 我的技能
  - 自我评价
  - 附加消息
- 实测详情页不依赖滚动懒加载主体内容。

## 项目经历展开

- 详情页可能默认只展示部分项目经历。
- 页面会显示类似：`项目经历（共20段）`。
- 默认 DOM 中可能只有 2 段项目。
- 展开入口：
  - `.rd-info-other-link`
  - 文案：`显示其他18段项目经历`
- 点击展开后，DOM 中项目经历数量变成完整数量。
- 抓取完整简历前必须先自动点击这个安全展开项。

## 附件简历

- 页面可能提示：
  - `附件简历包含更丰富的工作经历内容`
  - `索要附件`
  - `TA上传了附件简历，点击开聊向TA索要`
  - `向TA索要`
- `索要附件` 和 `向TA索要` 可能触发沟通或权益消耗。
- 系统不应自动点击。
- 正确处理：
  - 抓取当前可见正文。
  - 标记 `needs_attachment`。
  - 在提醒中心提示用户人工决策。

## 沟通与外部动作

- `立即沟通`、`查看联系方式`、`索要附件`、`向TA索要` 都属于高风险/消耗权益/对候选人产生影响的动作。
- 系统默认 dry-run。
- 自动发送应由任务策略控制，并受每日上限、分数阈值、账号状态、异常状态约束。

## 系统设计要求

- 账号首次登录、验证码、短信、滑块、安全提醒：暂停任务并提醒用户。
- 正常找人、搜索、列表抓取、详情页打开、项目展开、简历正文抓取、评分、话术生成：默认自动执行。
- 候选人数据必须按岗位隔离。
- 任务必须有可恢复状态机。
- 每一步必须记录日志，包括：
  - 当前账号
  - 当前任务
  - 当前 URL
  - 抓取字数
  - 项目经历展开数量
  - 附件状态
  - 分数和判断
  - 是否暂停及暂停原因
